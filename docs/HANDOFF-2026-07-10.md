## Architecture (the important shift)

**Bank-first.** Staff maintain a **question bank per exam**; students **generate a fresh randomized test on demand** — no pre-building.
- Each exam has a **blueprint** (`mock_exams.blueprint` jsonb) = "the ruling": modules, durations, and how many questions to draw per part.
- `mock_generate_attempt(exam)` builds a per-attempt **hidden** test (`is_generated=true`) from the active bank per the blueprint, reusing all existing timer/scoring/grading logic.
- Multi-exam is fully split by `exam_id` (questions, scoring bands, blueprint). Adding IELTS/MUET/TOEFL = data + renderers only → see **`docs/adding-an-exam.md`**. Greyed "Coming soon" panels already show on the student home.

## Student experience

- Home: **Start practice test** (auto-generates + resumes in-progress) per exam; optional curated tests listed separately; **progress-trend** chart across attempts; greyed IELTS/MUET/TOEFL panels.
- Test engine: server-authoritative timers, autosave (via `mock_save_response` RPC), question navigator, **Section X of N** stepper, **pre-submit review** of unanswered questions.
- All 5 APTIS modules with faithful renderers; **randomized dropdown order**; drag-and-drop ordering task.
- **Listening audio is AI-generated on demand** from the transcript (`/api/tts`, cached in `mock-media`) — no recording needed; multi-voice by speaker cue.
- **Speaking**: mic warm-up (practice, unsaved) → staged, timed, no-re-record capture → uploads to private `mock-speaking`.
- Result page: animated CEFR bar chart, per-skill explanations, **AI analysis** (strengths/weaknesses/recommendations + progress vs last attempt), teacher feedback, and a **print candidate report** (APTIS-style: scale /50, Final Scale /200, G&V separate, CEFR profile, skill descriptors).

## Staff experience (dashboard)

Nav grouped: `Overview · Grading | Question bank · Tests | Students · Settings`, active tab tinted crimson.
- **Question bank:** JSON import (multi-file), form **wizard for all 20 types** (add/edit), inline media upload + preview, **Preview-as-student**, search/filter, pagination, bulk delete/retire, delete-by-set-tag. **Listening audio** + **Speaking images** batch generators (OpenAI TTS / images; images are low-quality JPEG for cost).
- **Tests:** optional curated builder (auto-fill to APTIS targets, per-section fill counters, publish, delete). Generated tests are hidden here.
- **Grading queue:** "ready to verify" once all manual sections graded; workspace shows **full score summary (all sections)**, AI writing suggestion, speaking playback + prompt images, marking **rubric**, and one-click **Verify & release** (atomic `mock_verify_and_release` → also generates AI analysis).
- **Students:** search; per-student detail with mock attempts + read-only placement results; **remove attempt**.
- **Settings (admin):** create student/teacher/admin accounts (`mock_admin_create_user` — no service key), staff list, **scoring-band editor** (0–50 → CEFR per module).

## Known issues / watch-list

1. **Generated tests accumulate** one row per attempt (hidden). Harmless now; add a periodic GC of abandoned ones later.
2. **Blueprint editing** is DB-only (`mock_exams.blueprint`). A visual blueprint editor in Settings would be a nice follow-up (alongside the scoring-band editor).
3. **Listening TTS on Vercel Hobby** (10s function cap): a 4-voice speaker-matching item may be slow on first generation. Pre-warm on the "Listening audio" page, or the Team plan (60s) is fine.
4. **Scoring bands are provisional** placeholders (official BC conversion tables aren't public) — recalibrate in Settings when you have real data.
5. **Legacy placement tables** (`users`, `students`, etc.) have RLS disabled (pre-existing, out of scope). Sensitive columns (`passport_id`) are exposed to the anon key — a hardening pass on that repo someday, but do **not** enable RLS there without recreating its policies.

## Quick test checklist (live site)

1. Sign in as **student** → **Start practice test** → confirm a full randomized test opens (no 404), take a section, watch autosave say "saved".
2. Speaking: warm-up records + plays back; real questions record + upload.
3. Sign in as **staff** → **Grading** → open the attempt → score writing/speaking → **Verify & release**.
4. Back as student → **view result** → animated chart + AI analysis + **Print** report.
5. **Settings** → create a test account; **scoring-band editor** loads.
6. Toggle **dark mode** (header). Start a second attempt → **progress trend** appears on home.
